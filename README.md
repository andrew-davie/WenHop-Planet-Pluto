# WenHop: Planet Pluto

An Atari 2600 homebrew game built on the CDFJ+ ARM-coprocessor bankswitching
scheme (Harmony/Melody cart hardware). Gameplay logic runs as C on an ARM
co-processor; the 6502 side (DASM assembly) handles display/TIA timing and
calls back into the ARM code each frame.

Based on the CDFJ+ template (Chris Walton, Fred Quimby, Darrell Spice Jr,
2017), with additions by Craig Daniels (Gamax Software) and Andrew Davie.

## Building

**Run `make` from this directory (the repo root) — not from `main/`.**
Some paths in the Makefile (the bare `cdfj+_template.asm` reference, plus
`gfx/` and `tools/` sources) are only correct relative to the repo root.

```
make            # full build: tools, ROM, then launches the emulator
make clean      # remove build artifacts (keeps ROMs/ and screenshots/)
```

### External dependencies (siblings of this folder)

The Makefile expects these to exist one level above this repo, i.e. as
siblings of `WenHop-Planet-Pluto/` — edit the paths at the top of the
`Makefile` if yours live elsewhere:

- `../dasm/bin/dasm` — the DASM 6502 assembler
- `../Gopher2600/` — the Gopher2600 emulator (used for `EMULATOR=gopher`)
- `../../ArmGNUToolchain/15.2.rel1/...` — the arm-none-eabi GCC toolchain
- `/Applications/Stella.app` — optional, for `EMULATOR=stella`

### Build stages

The build is staged because the DASM (6502) side and the ARM (C) side each
need output from the other:

1. **prep** — create output folders, stub in a dummy ARM binary
2. **bootstrap_defines** — run DASM once against the dummy binary just to
   generate `main/defines_dasm.h` (the DASM symbol table as C `#define`s)
3. **arm** — compile/link the real ARM C code into the real ARM binary
4. **final** — run DASM again, now with the real ARM binary, to produce the
   final 6502 ROM

`make -j` is intentionally not used across these stages (see `.NOTPARALLEL`
in the Makefile) since each stage depends on output from the previous one.

## Layout

```
main/            C source for game logic, gameState_* screens, drawing,
                 sound, cave/board simulation, etc. (see main/*.h for the
                 module list)
main/custom/     Board-specific boot/link glue (custom.S, custom.boot.lds)
main/skull/      Standalone sub-build for the skull easter-egg graphics
main/spinningGlobe/  Procedurally-generated planet textures (C + the Python
                 generator scripts that produced them)
main/tools/      Small native build helpers (c_start, wav2raw2600)
kernels/         Per-gameState 6502 display kernels (DASM)
gfx/             Font/character-set source images and their generated C
gfx/tools/       One-off Python helper for the alphanumeric font
tools/           Python image-conversion scripts, plus the ~15 demo
                 "portrait" images/sources compiled into the ROM
icc/, bin/, output/  Generated at build time — safe to delete
ROMs/            Timestamped ROM snapshots from every build (gitignored,
                 kept locally as a build history/archive)
screenshots/     Emulator screenshots (gitignored)
doc/todo.txt     Running task list, not build documentation
```

Note: `tools/`, `main/tools/`, and `gfx/tools/` are three different,
unrelated directories that happen to share a name — worth double-checking
which one you mean when editing.

## Repo hygiene

- Don't commit `.venv/`, `__pycache__/`, or `.idea/` — the various Python
  helper scripts (`tools/`, `gfx/tools/`, `main/spinningGlobe/`) each have
  their own virtualenv; all three are gitignored.
- `main/defines_dasm.h` is generated by the build (stage 2 above) — never
  hand-edit it.
- `main/planetHuffman.c`/`.h` are generated from `main/planetInfo.c` — edit
  the latter, not the former.
